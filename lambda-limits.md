### Ограничения при работе с Лямбдами

# Лимиты AWS Lambda

| Ресурс / Параметр | Лимит | Комментарии |
|-------------------|--------|-------------|
| **Память (Memory)** | **128 MB – 10,240 MB** | Память влияет на CPU и сеть. |
| **Макс. время выполнения (Timeout)** | **15 минут (900 сек)** | Жёсткий лимит, увеличить нельзя. |
| **Ephemeral Storage (/tmp)** | **512 MB – 10,240 MB** | Настраиваемый, очищается между контейнерами. |
| **Развёрнутый пакет (uncompressed)** | **250 MB** | Включая слои. |
| **ZIP-пакет (compressed)** | **50 MB** | Лимит при загрузке ZIP напрямую. |
| **Container Image** | **10 GB** | Если деплоить Lambda как контейнер. |
| **Одновременные выполнения (Concurrency)** | **1,000 (soft limit)** | Можно увеличить через поддержку AWS. |
| **Reserved concurrency** | **0 – любое значение в рамках лимита аккаунта** | Гарантирует выделенные слоты. |
| **Provisioned concurrency** | Зависит от региона / квот | Гарантирует холодные старты = 0. |
| **Размер переменных окружения** | **4 KB** | Суммарный размер всех env-переменных. |
| **Synchronous invocation payload** | **6 MB** | API Gateway / SDK вызовы. |
| **Asynchronous event payload** | **256 KB** | S3, SNS, EventBridge и др. |
| **Максимум слоёв (Layers)** | **5** | До 5 слоёв одновременно. |
| **Threads / Processes / File descriptors** | **1,024** | Ограничение внутри runtime-контейнера. |
| **Количество точек монтирования EFS** | **1** | Можно подключить одну файловую систему. |
